## 2. Objetivo del proyecto

Construir una ruta tecnica para que Weib Player consuma una guia EPG util a partir de una playlist IPTV Xtream, aun cuando el proveedor entregue metadatos EPG incompletos o inconsistentes. El objetivo operativo es generar un XMLTV personalizado cuyos canales tengan nombres y/o identificadores compatibles con los canales visibles en la playlist, publicarlo mediante una URL HTTP/HTTPS y validar si Weib Player puede asociar la programacion a los canales del usuario.

## 5. Resumen ejecutivo

La investigacion confirma que el problema de EPG no se explica por una unica causa superficial. La playlist IPTV local contiene 9.883 entradas, pero 9.419 tienen `tvg-id=""`. En paralelo, el archivo `xmltv.xml` local contiene 9.859 nodos `channel`, pero 0 nodos `programme`; por tanto, ese XMLTV no puede alimentar una guia real aunque declare canales. Esta combinacion explica por que un reproductor puede detectar estructura EPG sin lograr asignar programacion util.

Se construyo una alternativa tecnica: `scripts/build_custom_xmltv.py` toma una fuente XMLTV con programacion y un CSV de matching, clona canales/programas por alias y genera IDs sinteticos `weib.<row>.<slug>.<sha1>`. La salida completa `output/custom_weib_epg.xml` contiene 1.818 canales y 161.419 programas, y su version `.gz` parsea correctamente. Esto prueba que la generacion XMLTV personalizada es viable.

El CSV de matching contiene 9.883 filas, 1.850 coincidencias aceptadas y 835 `tvg_id` fuente unicos. El README del paquete declara que la v1 usa solo coincidencias automaticas de confianza alta para evitar programacion incorrecta. Esta decision es correcta: fuerza precision antes que cobertura y evita contaminar Weib Player con asociaciones falsas.

No hay evidencia local persistida de respuestas `player_api`, `get_live_streams`, `get_short_epg`, capturas de Weib Player, configuracion GitHub Pages ni descarga HTTP desde Pages. Esas pruebas pudieron haberse realizado durante la conversacion previa, pero no quedaron en el workspace. Deben tratarse como hallazgos no reproducibles hasta que se vuelvan a capturar en archivos sanitizados.

El siguiente paso prioritario es ejecutar una prueba controlada de asociacion Weib con una playlist minima y un XMLTV minimo publicado por HTTPS: 3 a 5 canales, IDs y nombres deliberadamente alineados, y resultado documentado con captura o log. Esa prueba debe definir si Weib empareja por `tvg-id`, por `tvg-name`, por `display-name`, por nombre visible o por otra heuristica.

## 17. Lecciones aprendidas

### No volver a hacer

| Leccion | Justificacion |
|---|---|
| No asumir que un XMLTV con canales tiene EPG util | `xmltv.xml` tiene 9.859 canales y 0 programas |
| No depender de `tvg-id` de proveedor sin auditoria | 95% aproximado de entradas tienen `tvg-id` vacio |
| No usar XML compacto sin verificar `display-name` | La primera variante compacta perdio nombres visibles |
| No cerrar una integracion Weib sin captura/log persistido | El resultado externo queda no reproducible |
| No imprimir URLs completas de playlists IPTV | Pueden incluir dominios, tokens o credenciales |

### Funciono

| Practica | Resultado |
|---|---|
| Generacion por dos pasadas XMLTV | Maneja archivos grandes sin cargar todo el arbol completo |
| IDs sinteticos con hash | Evitan colisiones simples y dependencia del proveedor |
| CSV de matching con metodo y confianza | Hace auditable por que un canal entra o queda fuera |
| Variante comprimida `.gz` | Reduce 114.7 MB a 8.5 MB |
| Separar XML completo y compacto | Permite depurar compatibilidad y performance por separado |

### Acelerara futuras sesiones

| Acelerador | Uso |
|---|---|
| Script de conteo XMLTV/M3U | Validar rapidamente canales, programas, `tvg-id` y nombres |
| Evidencia sanitizada de Xtream API | Evitar repetir llamadas y proteger credenciales |
| Playlist minima de laboratorio | Determinar matching real de Weib sin ruido |
| Publicacion reproducible | Eliminar incertidumbre GitHub Pages/hosting |

## 22. Recomendaciones estrategicas

No limitar la evolucion a Weib Player. El hallazgo principal es mas amplio: muchos proveedores IPTV entregan playlists con metadatos EPG pobres, inconsistentes o incompletos. La oportunidad tecnica es convertir este proyecto en una herramienta profesional de auditoria de proveedores IPTV.

Recomendaciones:

| Recomendacion | Valor |
|---|---|
| Construir auditoria M3U/XMLTV independiente de reproductor | Permite diagnosticar proveedor antes de probar players |
| Mantener Weib como caso de compatibilidad | Sirve para validar integracion real de usuario |
| Generar reportes sanitizados | Facilita compartir evidencia sin filtrar secretos |
| Medir cobertura EPG por grupo/pais/categoria | Identifica brechas comerciales del proveedor |
| Separar matching automatico, heuristico y manual | Reduce falsos positivos |
| Incorporar multiples fuentes EPG | Aumenta cobertura y resiliencia |
| Agregar fixtures de reproductores | Comparar Weib, TiviMate, IPTV Smarters, VLC/Kodi u otros |
